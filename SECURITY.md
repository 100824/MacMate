# Security Policy

## Supported versions

Security fixes are handled for the latest source version on the default branch.

## Reporting a vulnerability

Please open a private security advisory on GitHub if available. If the repository host does not support private advisories, contact the maintainer directly before publishing exploit details.

When reporting, include:

- affected MacMate version or commit
- macOS version
- reproduction steps
- expected and actual behavior
- whether any local data, clipboard content, or API credentials may be exposed

## Secrets

Do not open issues or pull requests containing real API keys, tokens, private keys, or clipboard/private text samples. Replace them with placeholders such as `sk-REDACTED`.
